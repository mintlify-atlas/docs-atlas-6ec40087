# Installation Guide

This guide covers everything you need to install, configure, and run DevTools Hub API locally or with Docker.

## System Requirements

Before installing DevTools Hub API, ensure your system meets these requirements:

- **Node.js** ≥ v18.0.0
- **pnpm** package manager
- **PostgreSQL** ≥ 15 (or use Docker)
- **Docker & Docker Compose** (optional, for containerized setup)
- **Gmail account** with App Password (for email notifications)

<Note>
  While Docker is optional, it's the recommended way to run the API in development as it handles database setup automatically.
</Note>

## Local Installation

<Steps>
  <Step title="Clone the Repository">
    First, clone the DevTools Hub API repository from GitHub:

    ```bash
    git clone -b develop https://github.com/carlostajandev/devtools-hub-api.git
    cd devtools-hub-api
    ```

    <Note>
      We're cloning the `develop` branch which contains the latest stable development version.
    </Note>
  </Step>

  <Step title="Install Dependencies">
    Install all required npm packages using pnpm:

    ```bash
    pnpm install
    ```

    This will install all dependencies including:
    - **NestJS** framework components
    - **TypeORM** for database operations
    - **Passport** and **JWT** for authentication
    - **Nodemailer** for email notifications
    - **Helmet** for security headers
    - **Swagger** for API documentation

    <Warning>
      Make sure you have `pnpm` installed globally. If not, install it with: `npm install -g pnpm`
    </Warning>
  </Step>

  <Step title="Configure Environment Variables">
    Copy the example environment file and configure it with your settings:

    ```bash
    cp .env.example .env
    ```

    Edit the `.env` file with your configuration. See the [Environment Variables](#environment-variables) section below for detailed information.
  </Step>

  <Step title="Set Up PostgreSQL Database">
    Create a PostgreSQL database for the application:

    ```bash
    psql -U postgres
    CREATE DATABASE db_hub;
    \q
    ```

    Make sure your database credentials in `.env` match your PostgreSQL setup:

    ```bash .env
    DB_HOST=localhost
    DB_PORT=5432
    DB_USER=postgres
    DB_PASS=your_password
    DB_NAME=db_hub
    ```

    <Note>
      TypeORM will automatically create the necessary tables when you start the application.
    </Note>
  </Step>

  <Step title="Start the Development Server">
    Run the application in development mode with hot-reload:

    ```bash
    npm run start:dev
    ```

    You should see output indicating the server has started:

    ```
    🚀 Servidor corriendo en: http://localhost:3000/api
    📘 Documentación disponible en: http://localhost:3000/api/docs
    ```

    <Note>
      The API runs on port 3000 by default with the `/api` prefix for all routes.
    </Note>
  </Step>

  <Step title="Verify Installation">
    Test that the API is running correctly:

    ```bash
    curl http://localhost:3000/api/plans/all
    ```

    You should receive a JSON response (likely an empty array if no plans exist yet).

    Open Swagger documentation in your browser:
    
    👉 **[http://localhost:3000/api/docs](http://localhost:3000/api/docs)**
  </Step>
</Steps>

## Docker Setup

Using Docker Compose is the fastest way to get started as it handles both the API and database setup automatically.

<Steps>
  <Step title="Clone the Repository">
    ```bash
    git clone -b develop https://github.com/carlostajandev/devtools-hub-api.git
    cd devtools-hub-api
    ```
  </Step>

  <Step title="Configure Environment Variables">
    Create your `.env` file from the example:

    ```bash
    cp .env.example .env
    ```

    For Docker, you can use these database settings:

    ```bash .env
    DB_HOST=postgres
    DB_PORT=5432
    DB_USER=postgres
    DB_PASS=1234
    DB_NAME=db_hub
    ```

    <Note>
      The `DB_HOST` should be `postgres` (the Docker service name) instead of `localhost` when running in Docker.
    </Note>
  </Step>

  <Step title="Start Docker Containers">
    Launch all services using Docker Compose:

    ```bash
    docker-compose up -d
    ```

    This will start three containers:
    - **postgres** - PostgreSQL 15 database (port 5432)
    - **pgadmin** - PostgreSQL admin interface (port 5050)
    - **api** - DevTools Hub API (port 3000)

    <Note>
      The `-d` flag runs containers in detached mode (background).
    </Note>
  </Step>

  <Step title="View Logs">
    Monitor the API logs to ensure everything started correctly:

    ```bash
    docker-compose logs -f api
    ```

    Press `Ctrl+C` to exit log viewing.
  </Step>

  <Step title="Access Services">
    Your services are now available at:

    - **API:** http://localhost:3000/api
    - **Swagger Docs:** http://localhost:3000/api/docs
    - **pgAdmin:** http://localhost:5050
      - Email: `admin@devtools.com`
      - Password: `admin123`
  </Step>
</Steps>

### Docker Commands Reference

<CodeGroup>

## Environment Variables

DevTools Hub API requires several environment variables to be configured. Here's a complete reference:

### Database Configuration

PostgreSQL connection settings:

```bash .env
# Database host (use 'postgres' for Docker, 'localhost' for local)
DB_HOST=localhost

# Database port
DB_PORT=5432

# Database username
DB_USER=postgres

# Database password
DB_PASS=your_password

# Database name
DB_NAME=db_hub
```

### Email Configuration

Nodemailer settings for Gmail:

```bash .env
# SMTP host (Gmail)
MAIL_HOST=smtp.gmail.com

# SMTP port (465 for SSL)
MAIL_PORT=465

# Your Gmail address
MAIL_USER=your_email@gmail.com

# Gmail App Password (NOT your regular password)
MAIL_PASS=your_app_password

# Sender name and email
MAIL_FROM="DevToolsHub <your_email@gmail.com>"
```

<Warning>
  **Important:** You must use a Gmail App Password, not your regular Gmail password. See the [Gmail Setup](#gmail-setup-for-notifications) section below.
</Warning>

### Optional Configuration

```bash .env
# Environment (development, production)
NODE_ENV=development
```

## Gmail Setup for Notifications

DevTools Hub sends real email notifications using Nodemailer and Gmail. Follow these steps to configure Gmail:

<Steps>
  <Step title="Enable 2-Factor Authentication">
    1. Go to your [Google Account Settings](https://myaccount.google.com/)
    2. Navigate to **Security**
    3. Enable **2-Step Verification** if not already enabled

    <Note>
      App Passwords are only available when 2FA is enabled on your Google account.
    </Note>
  </Step>

  <Step title="Create an App Password">
    1. Visit [Google App Passwords](https://myaccount.google.com/apppasswords)
    2. Select **Mail** as the app
    3. Select **Other (Custom name)** as the device
    4. Enter "DevTools Hub API" as the name
    5. Click **Generate**
    6. Copy the 16-character password (shown without spaces)

    <Warning>
      Save this password immediately - you won't be able to see it again!
    </Warning>
  </Step>

  <Step title="Configure Environment Variables">
    Add the App Password to your `.env` file:

    ```bash .env
    MAIL_USER=your_email@gmail.com
    MAIL_PASS=abcd efgh ijkl mnop
    MAIL_FROM="DevToolsHub <your_email@gmail.com>"
    ```

    <Note>
      You can include or omit spaces in the App Password - both formats work.
    </Note>
  </Step>

  <Step title="Test Email Notifications">
    Register a new user to test email sending:

    ```bash
    curl -X POST http://localhost:3000/api/auth/register \
      -H "Content-Type: application/json" \
      -d '{
        "name": "Test User",
        "email": "test@example.com",
        "password": "password123"
      }'
    ```

    Check that the welcome email was received.
  </Step>
</Steps>

## Database Configuration

### TypeORM Settings

The API uses TypeORM for database management with the following configuration:

- **Auto-sync:** Enabled in development (creates/updates tables automatically)
- **Logging:** SQL queries are logged in development mode
- **Migrations:** Not required for development (auto-sync handles schema)

### Database Tables

TypeORM automatically creates these tables:

| Table | Description |
|-------|-------------|
| `users` | User accounts and authentication |
| `plans` | Subscription plans |
| `subscriptions` | User subscriptions to plans |
| `payments` | Payment records and transactions |
| `notifications` | Email notification logs |

### Manual Database Access

<CodeGroup>
  ```bash Local PostgreSQL
  psql -U postgres -d db_hub
  ```

  ```bash Docker PostgreSQL
  docker-compose exec postgres psql -U postgres -d db_hub
  ```
</CodeGroup>

## Troubleshooting

### Common Issues

<AccordionGroup>
  <Accordion title="Database connection failed">
    **Error:** `ECONNREFUSED` or `Connection refused`

    **Solutions:**
    - Verify PostgreSQL is running: `pg_isready`
    - Check database credentials in `.env`
    - For Docker: ensure `DB_HOST=postgres` not `localhost`
    - Check firewall isn't blocking port 5432
  </Accordion>

  <Accordion title="Email not sending">
    **Error:** `Invalid login` or `Authentication failed`

    **Solutions:**
    - Verify you're using an App Password, not your regular Gmail password
    - Check 2FA is enabled on your Google account
    - Ensure no extra spaces in MAIL_PASS
    - Try generating a new App Password
  </Accordion>

  <Accordion title="Port already in use">
    **Error:** `EADDRINUSE: address already in use :::3000`

    **Solutions:**
    - Stop other applications using port 3000
    - Find and kill the process: `lsof -ti:3000 | xargs kill -9`
    - Change the port in `src/main.ts` if needed
  </Accordion>

  <Accordion title="pnpm command not found">
    **Error:** `pnpm: command not found`

    **Solution:**
    ```bash
    npm install -g pnpm
    ```
  </Accordion>
</AccordionGroup>
