## Overview

This endpoint allows new users to register an account in the DevTools Hub platform. Upon successful registration, a welcome email is automatically sent to the user's email address.

<Note>
  The password is hashed using bcrypt with 10 salt rounds before being stored in the database.
</Note>

## Request Body

<ParamField body="name" type="string" required>
  Full name of the user.
  
  **Validation:**
  - Cannot be empty
  - Error message: "El nombre es requerido"
  
  **Example:** `"Juan Pérez"`
</ParamField>

<ParamField body="email" type="string" required>
  Valid email address for the user account. Must be unique in the system.
  
  **Validation:**
  - Must be a valid email format
  - Error message: "El email debe ser una dirección válida"
  
  **Example:** `"juan.perez@ejemplo.com"`
</ParamField>

<ParamField body="password" type="string" required>
  User's password for authentication.
  
  **Validation:**
  - Minimum length: 6 characters
  - Error message: "La contraseña debe tener al menos 6 caracteres"
  
  **Example:** `"MiContraseñaSegura123"`
</ParamField>

## Response

<ResponseField name="success" type="boolean" required>
  Indicates whether the registration was successful.
</ResponseField>

<ResponseField name="message" type="string" required>
  Human-readable message describing the result.
  
  **Example:** `"User registered successfully"`
</ResponseField>

<ResponseField name="data" type="object">
  Contains the newly created user information.
  
  <Expandable title="data properties">
    <ResponseField name="id" type="string">
      Unique identifier for the newly created user.
    </ResponseField>
    
    <ResponseField name="email" type="string">
      Email address of the registered user.
    </ResponseField>
  </Expandable>
</ResponseField>

## Implementation Details

<Warning>
  Password hashing is performed using bcrypt with 10 salt rounds. Never store passwords in plain text.
</Warning>

### Process Flow

1. **Validation**: Request body is validated against class-validator decorators
2. **Email Check**: System verifies that the email is not already registered
3. **Password Hashing**: Password is hashed using bcrypt (10 salt rounds)
4. **User Creation**: User entity is created and saved to the database
5. **Welcome Email**: Confirmation email is automatically sent via `SendPaymentConfirmationUseCase`
6. **Response**: Returns user ID and email upon success
